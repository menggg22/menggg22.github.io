Personal Knowledge Operating System with LLM

4 months ago I started building a personal system with an LLM. Somewhere along the way it became a knowledge operating system — it ingests what I learn, synthesizes patterns over time, plans my week, and surprises me with what I can build.

It has two components: a file structure where knowledge lives, and a set of skills, reusable LLM workflows that operate on that structure. Together they form a loop with four phases: Ingest brings knowledge in, Synthesize finds patterns, Act drives decisions, and Compound turns actions into new knowledge that feeds back to Ingest.

  ┌──────────────────────────────────────────────────────────┐
  │                                                          ↓
Ingest                Synthesize             Act             Compound
/learn                /monthly-review        /journal plan   actions create
/innovate add         /write-blog            /journal review new knowledge
/journal log          /learning-quarterly    /innovate work  → back to Ingest
                                             /play
  │                         │                    │               │
  ↓                         ↓                    ↓               ↓
KNOWLEDGE/            JOURNAL/reviews/       JOURNAL/         KNOWLEDGE/
  learning-*          PUBLISHED/             FUN/sparks       new learning-*
  idea-*                                                      new project-*/

Ingest

Raw knowledge comes in.

/learn — I give it a URL or describe what I read. The LLM reads the material, we discuss it, and it files the insights into the right topic folder as a learning-* entry. Not a bookmark — a processed, discussed piece of knowledge.

/innovate add — I describe a half-formed idea. The LLM evaluates it — feasibility, effort, who would use it. We debate. It files the result as an idea-* entry. Some ideas die here. Some become project-*/ folders.

/journal log — Quick capture after a work session. What I did, how long, any insights. Appends to the monthly log.

The naming convention does the organizing: learning-*, idea-*, blog-*, project-*. The LLM searches for these patterns. No database, no index. The filesystem is the query engine.

Synthesize

Knowledge on its own is noise. Synthesis turns it into signal.

/monthly-review — Reads everything from the month: journal entries, learning files, blog drafts, project progress, git history. Finds patterns I didn’t see. “You spent 3 weeks going deep on X but your stated goal is Y. Intentional?” Writes a review to JOURNAL/reviews/.

/write-blog — Tracks a post from idea → outline → draft → publish. Lives in KNOWLEDGE/ as a blog-*/ folder while drafting, moves to PUBLISHED/ when shipped. Writing is compression — it forces scattered learnings into a coherent argument.

/learning-quarterly — Reads 3 months of monthly reviews. Asks: am I heading where I want to go? Is what I’m learning in demand? Course correct or stay?

The temporal structure matters. Weekly → monthly → quarterly. Each layer reads the output of the layer below. January’s review noticed I was building expertise but sharing nothing. That shaped February — I published 3 posts. March pushed toward external publishing. Each review feeds the next.

Act

/journal plan — Every Monday, reads last week’s review, active projects, learning gaps. Generates a weekly plan with concrete success criteria. Not generic advice — a plan that knows what slipped last week and what matters this week.

/journal review — Friday. Pulls git data, checks success criteria, asks reflection questions. Writes the review that next Monday’s plan will read.

/innovate work — Pick a project and work on it. Loads context from the project-*/ folder, picks up where you left off.

/play — Saturday night. Picks two random topics from the knowledge base, finds a connection I didn’t see, and we explore for an hour. No pressure, no goals. Just curiosity with a safety net.

Compound

Actions generate new knowledge. The loop closes.

/play sparked an idea: simulate a 19th century utopian community with AI agents. 36 hours later I had a working system — 5 agents with historical personas, a 3-phase lifecycle (Act → Talk → Reflect), 6 counterfactual experiments. The agents reproduced real historical outcomes without being told what happened.

That project went back into KNOWLEDGE/ as new learnings about agent architecture. Those learnings feed the next /journal plan. The plan shapes the week. The week generates new entries.

A learning session on agent architectures in Week 3 became a shipped project in Week 8. Not because I planned it — because the system compounds.

What’s Missing

  • Visual exploration — no backlinks, no graph view. At scale, discovery breaks.
  • Portability — tightly coupled to one LLM tool. Architecture is portable, implementation isn’t.
  • Single user — deeply personal. Generalizing might kill what makes it work.

Try It

Clone the template repo and make it yours. Works with any LLM CLI that reads CLAUDE.md.

Written by a human and her LLM agent, during a /play session that started with “let’s do something fun tonight” and ended with the system reflecting on its own architecture. The knowledge OS built this blog post about itself. We think that counts as compound.
